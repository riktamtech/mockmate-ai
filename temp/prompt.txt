### Frontend Optimization (Mockmate-AI)

- **React:** `React.memo`, `useMemo`, `useCallback`, `lazy`+`Suspense` for all new pages/heavy components, code splitting per route.
- **Virtualization:** `react-window` or `react-virtual` for all long lists (job openings, notifications, analytics tables).
- **CSS:** CSS variables, no inline styles in render paths, GPU-composited animations only (`transform`, `opacity`), `will-change` where justified.
- **Animations:** Framer Motion (if already used) or CSS keyframes — 60fps, no layout thrash.
- **API:** Debounce search inputs (300ms), throttle scroll handlers, AbortController on unmount for all fetches, React Query or SWR for server state (match existing patterns).
- **Images/Assets:** Lazy load all images, WebP format, proper `srcset`.
- **Bundle:** Dynamic imports for all new pages.
- All new UI **must be fully responsive** across mobile, tablet, and desktop.

### Backend Optimization

- **DB Queries:** Compound indexes on `job_openings` (isEnabled, openingType, createdAt, orgId), `job_applications` (candidateId, openingId, status), `notifications` (userId, isRead, createdAt).
- **Pagination:** Cursor-based pagination (not offset) for infinite scroll endpoints.
- **Caching:** Redis cache for public openings list (TTL: 5 min, invalidated on sync event), fitness score semantic cache (TTL: 7 days).
- **API Design:** RESTful, versioned (`/v1/`), consistent error envelope `{ success, data, error, meta }`, rate-limited cross-server endpoints.

### Security

- Cross-server (Zinterview ↔ Mockmate) auth: **HMAC-signed requests** with shared secret + JWT service token + request timestamp (anti-replay, 5-min window).
- All recruiter action endpoints: permission middleware matching existing Zinterview RBAC patterns.
- All candidate-facing endpoints: candidate auth middleware from existing Mockmate auth patterns.
