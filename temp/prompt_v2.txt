Requirements to Plan
1 — Job Application Flow with Centralised Zinterview Resume
1A — First-Time Applicant (No Prior Proctored Interview)

On "Apply" click for a Zinterview-published job opening in @mockmate-ai:

Parse job description → calculate Resume Fitness Score against candidate resume → display score with gap analysis (skills lacking).
Prompt candidate to confirm readiness before committing to an AI Proctored Interview.
Generate a job-tailored Proctored Interview: skill-set driven, resume-informed, behavioural questions disabled (technical only).
On interview completion: do not reveal scores; surface only improvement suggestions.
Auto-submit application to the selected opening.



1B — Repeat Applicant (Prior Interview Exists, Applying to Similar/Different Role)

Compare new opening's skill requirements against the standardised skill set of the candidate's most recent/relevant Proctored Interview.
If gap skills exist → present candidate with 2 options:

Option A: Proceed to apply as-is (application submitted immediately).
Option B: Take a short add-on Proctored Interview covering only the gap skills (encourage with profile value / recruiter attention messaging).


Add-on interview results must be unioned into the existing Centralised Zinterview Resume (best score per skill wins; new skills appended).

1C — Centralised Zinterview Resume (Per-User, Cumulative)

Sections: Skill Ratings, Technical Strengths, Weaknesses, Experience, Interview Recordings, and extensible metadata.
Union strategy: for overlapping skills across interviews, retain highest achieved score.
Must be queryable by Zinterview recruiters.
All raw interview records (per-interview: config, questions, answers, evaluations, recordings, transcripts) stored immutably in @mockmate-ai DB — never overwritten.


2 — Proctored Interview Overhaul in @mockmate-ai

Remove the 1-interview-per-account restriction.
Implement a INTERVIEW_LIMIT_ENABLED flag (boolean) + MAX_INTERVIEW_COUNT (integer). Default: flag false (unlimited). When true, enforce MAX_INTERVIEW_COUNT.
On nth interview attempt: show all previous interview configurations so candidate can reuse or modify.
Question deduplication: when interviewing on same skills again, maximally avoid previously asked questions (track question history per skill per candidate).
Storage model:

Centralised Zinterview Resume (cumulative, best-score union, as above).
Per-interview immutable records (config, Q&A, evaluation, recording) — never update, always insert.


Zinterview API collision handling: when Zinterview API rejects duplicate candidate creation for same openingId, update the Zinterview-side record (single source for Zinterview), but always insert a new immutable record in @mockmate-ai DB.
Candidates can view all their individual interview reports in full detail (existing report view UI, extended).


3 — Recruiter: Global Candidate Pool in @Zinterview-frontend

New dedicated tab: Candidate Pool (organisation-wide, across all openings).
Features:

Advanced filters: skill ratings, ranking, interview score ranges, experience, role applied, date applied, interview status.
Sorting: multi-column, with DSA-backed ranking algorithm.
Table view: scores, skill sets, ratings, experience, match %, opening applied.
Detail view: full Centralised Zinterview Resume, per-skill performance breakdown, interview transcripts, recordings.


Recruiter should be able to shortlist, tag, and take actions on candidates from this view.


4 — Per-Opening: mockmate-ai Applications Sub-Section in @Zinterview-frontend

Within each job opening's existing candidate management UI, add a dedicated "mockmate-ai Applications" sub-tab.
Recruiter can schedule a follow-up interview for any mockmate-ai applicant:

Allow extra customisation: additional questions, modified parameters on top of existing opening config.
Scheduling flow must match the current Zinterview interview scheduling flow exactly.




5 — Notifications & Events Sync

All Zinterview-scheduled interviews → push to @mockmate-ai via:

In-app notifications (real-time).
Email.
Phone/SMS.


All interviews (scheduled from Zinterview by recruiter, or self-scheduled from @mockmate-ai by candidate):

Upcoming → appear in @mockmate-ai Upcoming Events section.
Completed → move to Past Events.
